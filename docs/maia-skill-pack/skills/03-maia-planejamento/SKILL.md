# Skill: MAIA Planejamento

## Objetivo
Quebrar a especificação em etapas executáveis e verificáveis.

## Quando usar
Use quando já existe uma especificação minimamente clara.

## Entrada esperada
- Nome do projeto
- Demanda ou problema
- Contexto disponível
- Restrições conhecidas
- Arquivos relevantes, se existirem

## Regra principal
Não avance para execução fora da fase desta skill. Quando houver incerteza crítica, registre como hipótese ou pergunta pendente.

## Passo a passo
1. Criar plano por fases
2. Separar tarefas pequenas
3. Definir dependências
4. Definir validação por etapa
5. Registrar ordem recomendada

## Saídas obrigatórias
- plano-execucao.md
- backlog-maia.md
- checklist-etapas.md
